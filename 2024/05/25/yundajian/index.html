在云端上搭hexo博客

一、前期准备

1.在相关网站购买云服务器

购买云服务器的地方有很多，像阿里云、华为云、腾讯云等等，具体选择综合考虑，我这里使用阿里云作演示。

选择操作系统镜像

建议选择centos7的

防火墙设置

注意把ssh服务（即22号端口）和http服务（80端口）打开

3.使用Xshell连接

购买完服务器后就可以远程连接了，这里推荐xshell远程连接；

账号和密码在购买服务器时便已设定了，这里说一下要用公网ip连接

之后选择属性，将用户名和密码填入即可登录

3.在本地搭建hexo

后面，这里不再赘述。以上的是前期准备基础。

二、登录操作

1.安装配置git

1.安装依赖包环境

yum install -y curl-devel expat-devel gettext-devel openssl-devel zlib-devel

2.安装编译工具

yum install -y gcc perl-ExtUtils-MakeMaker package

3.删除旧版本git

这里吐槽一下，为啥到现在了yum支持的git版本还是1.18的

yum remove git -y

4.进入家目录

cd

5.下载git到家目录

这里git选择2.34.0，如有更新，请自行到官网下载

wget https://www.kernel.org/pub/software/scm/git/git-2.34.0.tar.gz --no-check-certificate

如果没有wget命令，请安装：

yum install wget -y

6.解压文件

tar -zxf git-2.34.0.tar.gz

7.安装git

进入到git目录下

cd git-2.34.0

编译

make all prefix=/uer/local/git

安装git到/usr/local/git下

make install prefix=/usr/local/git

8.配置git环境变量

将git加入PATH目录中

echo 'export PATH=$PATH:/usr/local/git/bn' >> /etc/bashrc

使配置生效

source /etc/bashrc

查看git版本

git --version

2.配置用户并配置SSH免密登录

1.创建用户并设置密码

这里的username为你自己的用户名，并为用户名设置一个密码

adduser [username]
passwd [username]

2.为用户分配权限

usermod [uesrname] -G wheel

3.本机创建密钥

在本机电脑终端输入：

ssh-keygen -t rsa

然后一直回车，新建一个密钥

终端上会显示你密钥创建的路径，找到它（id_rsa.pub文件）并用笔记本打开，复制里面的内容。

4.配置公钥

回到服务器终端，切换用户

su [username]

回到家目录

cd

创建.ssh文件夹

mkdir .ssh

新建authorized_keys

vim .ssh/authorized_keys

如果没有vim，请安装：

sudo yum -y vim

将复制的公钥内容复制到authorized_keys中，退出保存

3.git仓库配置

1.退出当前用户

exit

2.此时为root用户，创建git目录，并修改目录所有权和用户权限

mkdir /home/git/
chown -R [username]:[passwd] /home/git/
chmod 755 /home/git/

3.新建新的钩子文件

vim /home/git/blog.git/hooks/post-receive

4.进入文本编辑器中，粘贴下面两行

#！/bin/bash
git --work-tree=/home/blog --git-dir=/home/git/blog.git checkout -f

5.保存退出，然后修改文件权限

chmod +x /home/git/blog.git/hooks/post-receive

4.配置nginx

1.首先创建blog目录用户转发

mkdir /home/blog/
chown -R[username]:[passwd] /home/blog/
chmod -R 755 /home/blog/

2.安装nginx

方法

1.搜索阿里云镜像

2.找epel镜像，输入到命令行中

3.输入命令安装nginx

yum install -y nginx

3.启动nginx

systemctl start nginx.service

4.查看nginx服务状态，绿色的active即成功

systemctl status nginx.service

5.查看nginx的默认配置文件的安装位置

nginx -t

查看其中

nginx: configuration file /etc/nginx/nginx.conf test is successful中的文件，这里是configuration file /etc/nginx/nginx.conf,每个人可能不一样

随后

vim configuration file /etc/nginx/nginx.conf

打开后找到下面位置并修改成

    server {
        listen       80;
        listen       [::]:80;
        server_name  [自己的公网ip或域名];
        root         /home/blog;

        # Load configuration files for the default server block.
        include /etc/nginx/default.d/*.conf;

        error_page 404 /404.html;
        location = /404.html {
        }

保存退出，重启nginx服务

systemctl restart nginx.service

5.在hexo站点配置文件中配置

打开本地hexo博客所在文件，打开配置文件_config.yml

找到并修改

deploy:
  type: git
  repository: [username]@[公网ip地址]:/home/git/blog.git
  branch: master

6.从本地上传hexo

hexo clean //清除本地缓存
hexo g //生成网站静态文件到默认设置的 public 文件夹
hexo d //生成静态网站，并部署到设定仓库

三、配置免密登录（可选）

检查.ssh权限

事先说明，这个免密登录我没有成功，不知道是哪个文件权限没给，如果哪位大佬知道，希望能指正。但是这个是可选项，没成功不影响使用

回到家目录

cd

到[username]目录下

cd /home/[username]

检查权限

修改权限

chmod 700 .ssh

到.ssh目录,配置authorized_keys权限

cd .ssh

修改权限

chmod 644 authorized_keys
